# webcore

#### 介绍
Webcore 是基于 原生 Web Components 标准设计的一套前端运行时与开发框架。

它围绕「服务化架构 + 组件系统 + 路由系统 + 事件系统」构建，为浏览器环境提供一套轻量但完整的前端基础设施，让你在不依赖大型三方框架的前提下，依然可以用统一、可维护的方式开发中小型 Web 应用。

#### 框架优势
1. 原生优先、长期可用
完全建立在浏览器原生能力之上，一次开发多处使用，不绑定任何大型框架生态。官方标准，长期支持。
天然支持样式隔离、热重载、模块化、远程加载、懒加载等。无需构建工具，不绑定工具链，调试容易。
完全基于原生能力的封装，不创造新语法，会原生开发即可上手，学习成本低。

2. 服务化架构，清晰可扩展
所有能力都抽象为服务：组件、路由、事件等，由服务容器统一管理和注入。
组件既是服务，各个组件可以独立开发，独立部署，独立维护，不用牵一发而动全身。
支持微服务，各个服务和组件互相独立，但又可通过接口相互通讯，服务之间，组件直接彻底解耦。
